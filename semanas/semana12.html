CONTENIDO DE LA SEMANA. Spring Core y Spring boot

Spring Boot facilita la creación de aplicaciones independientes basadas en Spring de nivel de producción que puede "simplemente ejecutar". Adoptamos una visión objetiva de la plataforma Spring y de las bibliotecas de terceros para que pueda comenzar con el mínimo esfuerzo. La mayoría de las aplicaciones Spring Boot necesitan una configuración mínima de Spring. Si busca información sobre una versión específica o instrucciones sobre cómo actualizar desde una versión anterior, consulte la sección de notas de la versión del proyecto en nuestra wiki. Características Cree aplicaciones Spring independientes Incruste Tomcat, Jetty o Undertow directamente (no es necesario implementar archivos WAR) Proporcione dependencias 'iniciales' obstinadas para simplificar la configuración de su compilación Configure automáticamente bibliotecas Spring y de terceros siempre que sea posible Proporcionar funciones listas para producción, como métricas, comprobaciones de estado y configuración externalizada. Absolutamente sin generación de código y sin necesidad de configuración XML Empezando Súper rápido: prueba la Guía de inicio rápido . Más general: intente crear una aplicación con Spring Boot Más específico: intente crear un servicio web RESTful . O busque en todas nuestras guías en la página de inicio de Guías . Spring Core Esta parte de la documentación de referencia cubre todas las tecnologías que son absolutamente integrales para Spring Framework. El más destacado de ellos es el contenedor de inversión de control (IoC) de Spring Framework. Un tratamiento exhaustivo del contenedor IoC de Spring Framework es seguido de cerca por una cobertura integral de las tecnologías de programación orientada a aspectos (AOP) de Spring. Spring Framework tiene su propio marco AOP, que es conceptualmente fácil de entender y que aborda con éxito el punto óptimo del 80 % de los requisitos de AOP en la programación empresarial Java. También se proporciona cobertura de la integración de Spring con AspectJ (actualmente la implementación AOP más rica, en términos de características, y ciertamente la más madura en el espacio empresarial de Java). El procesamiento AOT se puede utilizar para optimizar su aplicación con anticipación. Normalmente se utiliza para la implementación de imágenes nativas mediante GraalVM.
