¿ Cuándo debo utilizar getServerSideProps? Debe utilizarlo getServerSidePropssi necesita representar una página que se base en datos de usuario personalizados o información que solo se puede conocer en el momento de la solicitud. Por ejemplo, authorizationcabeceras o una geolocalización. Si no necesita recuperar los datos en el momento de la solicitud o prefiere almacenar en caché los datos y el HTML preprocesado, le recomendamos utilizar getStaticProps. Comportamiento getServerSidePropsse ejecuta en el servidor. getServerSidePropsSólo se puede exportar desde una página . getServerSidePropsdevuelve JSON. Cuando un usuario visita una página, getServerSidePropsse utilizará para recuperar datos en el momento de la solicitud y los datos se utilizarán para representar el HTML inicial de la página. propspasado al componente de la página se puede ver en el cliente como parte del HTML inicial. Esto es para permitir que la página se hidrate.correctamente. Asegúrese de no pasar ninguna información confidencial que no debería estar disponible en el cliente en props. Cuando un usuario visita la página a través next/linkde o next/router, Next.js envía una solicitud API al servidor, que ejecuta getServerSideProps. No es necesario llamar a una ruta API de Next.js para recuperar datos cuando se utiliza, getServerSidePropsya que la función se ejecuta en el servidor. En su lugar, puede llamar a un CMS, una base de datos u otras API de terceros directamente desde dentro getServerSideProps.
