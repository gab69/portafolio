Los Hooks te permiten usar diferentes funciones de React de tus componentes. Puedes usar los Hooks incorporados o combinarlos para construir los tuyos propios.

State Hooks

El estado permite que un componente "recuerde" información como la entrada del usuario. Por ejemplo, un componente de formulario puede usar el estado para almacenar el valor de entrada, mientras que un componente de galería de imágenes puede usar el estado para almacenar el índice de la imagen seleccionada.

Para agregar estado a un componente, use uno de estos Hooks:

- useState declara una variable de estado que puede actualizar directamente.
- useReducer declara una variable de estado con la lógica de actualización dentro de una función reductora.

El contexto permite que un componente reciba información de padres lejanos sin pasarla como accesorios. Por ejemplo, el componente de nivel superior de su aplicación puede pasar el tema de la interfaz de usuario actual a todos los componentes siguientes, sin importar cuán profundos sean.
useContext lee y se suscribe a un contexto.


Ref Hooks

Las referencias permiten que un componente contenga información que no se utiliza para la renderización, como un nodo DOM o un ID de tiempo de espera.
A diferencia del estado, actualizar una referencia no vuelve a representar su componente.
Las referencias son una «escotilla de escape» del paradigma React.
Son útiles cuando necesita trabajar con sistemas que no son React, como las API del navegador integradas.

useRef declara una referencia. Puede contener cualquier valor, pero la mayoría de las veces se usa para contener un nodo DOM.

useImperativeHandle le permite personalizar la referencia expuesta por su componente. Esto rara vez se usa.


Performance Hooks

Los efectos permiten que un componente se conecte y sincronice con sistemas externos. Esto incluye lidiar con la red, el DOM del navegador, animaciones, widgets escritos usando una biblioteca de UI diferente y otro código que no sea de React.

useEffect conecta un componente a un sistema externo.

Los efectos son una «escotilla de escape» del paradigma React. No utilice efectos para orquestar el flujo de datos de su aplicación. Si no estás interactuando con un sistema externo, es posible que no necesites un efecto.

Hay dos variaciones de useEffect que rara vez se utilizan con diferencias en el tiempo:

1.- useLayoutEffect se activa antes de que el navegador vuelva a pintar la pantalla. Puedes medir el diseño aquí.
2.- useInsertionEffect se activa antes de que React realice cambios en el DOM. Las bibliotecas pueden insertar CSS dinámico aquí.

Performance Hooks

Una forma común de optimizar el rendimiento de la renderización es omitir el trabajo innecesario. Por ejemplo, puede decirle a React que reutilice un cálculo almacenado en caché o que omita una nueva renderización si los datos no han cambiado desde la renderización anterior.
Para omitir cálculos y renderizaciones innecesarias, utilice uno de estos ganchos:

1.- useMemo le permite almacenar en caché el resultado de un cálculo costoso.
2.- useCallback le permite almacenar en caché una definición de función antes de pasarla a un componente optimizado.

A veces, no puedes omitir la reproducción porque la pantalla realmente necesita actualizarse. En ese caso, puede mejorar el rendimiento separando las actualizaciones bloqueantes que deben ser sincrónicas (como escribir en una entrada) de las actualizaciones sin bloqueo que no necesitan bloquear la interfaz de usuario (como actualizar un gráfico).

Para priorizar el renderizado, utilice uno de estos Hooks:

useTransition le permite marcar una transición de estado como sin bloqueo y permitir que otras actualizaciones la interrumpan.

useDeferredValue le permite posponer la actualización de una parte no crítica de la interfaz de usuario y dejar que otras partes se actualicen primero.

Resource Hooks

Un componente puede acceder a los recursos sin tenerlos como parte de su estado. Por ejemplo, un componente puede leer un mensaje de una Promesa o leer información de estilo de un contexto.

Para leer un valor de un recurso, utilice este hook:

Use permite leer el valor de un recurso como una Promesa o un contexto.


Other Hooks

These Hooks are mostly useful to library authors and aren’t commonly used in the application code.

1.- useDebugValue lets you customize the label React DevTools displays for your custom Hook.

2.- useId lets a component associate a unique ID with itself. Typically used with accessibility APIs.

3.- useSyncExternalStore lets a component subscribe to an external store.
